# AI Documentation Directory

This directory contains project tracking and planning documents used by AI assistants during development.

## Directory Structure

### `/project/`
**Purpose:** High-level project definition and status documents

- **PROJECT_DEFINITION.md** - Project vision, roadmap, architecture philosophy, and feature planning
- **PROJECT_RESUME.md** - Quick-start context for AI assistants; current project state snapshot

### `/tracking/`
**Purpose:** Development changelog and historical tracking

- **DEVELOPMENT.md** - Running index of all changes, organized by effort/phase (see `.cursorrules` Rule 1)

### `/planning/`
**Purpose:** Phase-specific planning documents and implementation notes

- Reserved for detailed planning documents created during effort/phase planning
- Examples: implementation strategies, API designs, technical spike notes

## Usage

**For AI Assistants:**
1. Read `PROJECT_RESUME.md` first for quick context
2. Check `PROJECT_DEFINITION.md` for long-term vision and architecture
3. Update `DEVELOPMENT.md` after each phase/effort (per `.cursorrules` Rule 1)

**For Humans:**
- `PROJECT_DEFINITION.md` - Understand project goals and roadmap
- `DEVELOPMENT.md` - Review what changed and when
- `PROJECT_RESUME.md` - Get up to speed quickly on current state

## Related Documents

- **`.cursorrules`** (root) - AI development rules and workflow
- **`docs/features/`** - Feature specifications and design documents
- **`docs/architecture/`** - Architecture decision records (ADRs)
